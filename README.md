# ETL Pipeline: API → PostgreSQL

Учебный ETL-пайплайн для портфолио. Загружает данные из публичного REST API, валидирует их и сохраняет в PostgreSQL.

## Что умеет этот пайплайн

- **Пагинация** — загружает данные по 20 записей, пока не закончатся
- **Retry-политика** — при ошибках повторяет запрос с экспоненциальной задержкой (2^attempt + jitter)
- **Валидация данных** — проверяет типы (int), обязательные поля (id, userId, title)
- **Сохранение в PostgreSQL** — через `executemany` для массовой вставки
- **Логирование** — все действия и ошибки пишутся в файл

## Стек технологий

- Python 3.10
- requests — HTTP-запросы к API
- psycopg2 — работа с PostgreSQL
- python-dotenv — переменные окружения
